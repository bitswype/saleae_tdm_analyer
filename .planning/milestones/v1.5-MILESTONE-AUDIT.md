## REQ-15 Fix Verification

Phase 11 corrected the accumulate-before-flush ordering bug identified in the first audit.

**Corrected decode() ordering:**
```python
self._try_derive_sample_rate(frame)   # 1. derive rate
self._try_flush(frame_num)             # 2. FLUSH (reads clean previous-frame accum)
if not (frame.data.get('short_slot') or frame.data.get('bitclock_error')):
    self._accum[slot] = _as_signed(...)  # 3. ACCUMULATE (into clean new-frame accum)
self._last_frame_num = frame_num
```

**Self-test confirmation:** `python3 hla-wav-export/TdmWavExport.py` outputs:
```
All self-tests passed.
REQ-15 decode ordering test passed.
```

Two-frame input `[TDM1: slot0=100, slot1=200] + [TDM2: slot0=300, slot1=400]` → WAV `[100, 200, 300, 400]` ✓
