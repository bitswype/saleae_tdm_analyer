**Constructor ŌĆö RF64WaveFileHandler::RF64WaveFileHandler(std::ofstream& file, U32 sample_rate, U32 num_channels, U32 bits_per_channel):**
- Same initializer list pattern as PCMWaveFileHandler (mFile, mSampleRate, mNumChannels, mBitsPerChannel, mSampleCount=0, mTotalFrames=0, mWtPosSaved=0, mSampleData=nullptr, mSampleIndex=0)
- Same bit-depth-to-bytes ladder (8/16/32/40/48/64) ŌĆö set `mRF64Header.mBitsPerSample` and `mBytesPerChannel`
- Set `mRF64Header.mBlockSizeBytes = mBytesPerChannel * num_channels`
- Set `mFrameSizeBytes = mRF64Header.mBlockSizeBytes`
- Set `mBitShift = mRF64Header.mBitsPerSample - bits_per_channel`
- Set `mRF64Header.mNumChannels = num_channels`
- Set `mRF64Header.mSamplesPerSec = sample_rate`
- Set `mRF64Header.mBytesPerSec = mFrameSizeBytes * sample_rate`
- Write the 80-byte header: `mFile.write((const char*)(&mRF64Header), sizeof(mRF64Header))`
- Allocate `mSampleData = new U64[mNumChannels]`

**Destructor ŌĆö ~RF64WaveFileHandler():**
- Same as PCMWaveFileHandler: call close() if file is open.

**addSample(U64 sample):**
- Identical to PCMWaveFileHandler::addSample ŌĆö same 8-bit offset encoding, same frame assembly, same writeLittleEndianData per channel, same periodic updateFileSize call every `mSampleRate / 100` frames.
- CRITICAL: `mTotalFrames++` and `mSampleCount += mNumChannels` are U64 operations (no overflow).

**writeLittleEndianData(U64 value, U8 num_bytes):**
- Identical copy of PCMWaveFileHandler::writeLittleEndianData. Same byte-by-byte little-endian write loop.

**close():**
- Call updateFileSize()
- Write odd-byte pad if `(mTotalFrames * mFrameSizeBytes) % 2 == 1` ŌĆö same as PCMWaveFileHandler. NOTE: use U64 multiplication to avoid overflow: `((mTotalFrames * (U64)mFrameSizeBytes) % 2)`
- Close file, delete[] mSampleData, set mSampleData = nullptr

**updateFileSize():**
- Save current position: `mWtPosSaved = mFile.tellp()`
- Compute data size: `U64 dataSizeBytes = mTotalFrames * (U64)mFrameSizeBytes;` then even-byte pad: `if ((dataSizeBytes % 2) == 1) dataSizeBytes += 1;`
- Compute riffSize: `U64 riffSize = 72ULL + dataSizeBytes;` (80 header - 8 for RF64+ckSize fields)
- Compute sampleCount: `U64 sampleCount = mTotalFrames * (U64)mNumChannels;`
- Seekback write three ds64 fields as 8-byte little-endian:
  - `mFile.seekp(RF64_DS64_RIFFSIZE_POS); writeLittleEndianData(riffSize, 8);`
  - `mFile.seekp(RF64_DS64_DATASIZE_POS); writeLittleEndianData(dataSizeBytes, 8);`
  - `mFile.seekp(RF64_DS64_SAMPLECNT_POS); writeLittleEndianData(sampleCount, 8);`
- Do NOT write to offset 4 (RIFF sentinel) or offset 76 (data sentinel) ŌĆö those stay 0xFFFFFFFF
- Restore position: `mFile.seekp(mWtPosSaved)`

IMPORTANT: The existing `writeLittleEndianData` already handles 8-byte writes correctly because it loops `num_bytes` times shifting 8 bits per iteration. No modification needed.
