**Restructure the export block with conditional dispatch (RF64-03):**
After the size estimation (which computes `estimated_data_bytes` and `WAV_MAX_DATA_BYTES`), open the file and branch:

```
std::ofstream f;
f.open(file, std::ios::out | std::ios::binary);

if (f.is_open())
{
    U16 num_slots_per_frame = mSettings->mSlotsPerFrame;

    if (estimated_data_bytes > WAV_MAX_DATA_BYTES)
    {
        // RF64 path for >4 GiB exports
        RF64WaveFileHandler wave_file_handler(f, mSettings->mTdmFrameRate,
                                               mSettings->mSlotsPerFrame,
                                               mSettings->mDataBitsPerSlot);
        for (U64 i = 0; i < num_frames; i++)
        {
            Frame frame = GetFrame(i);
            if (frame.mType < num_slots_per_frame)
            {
                U64 sample_value = (frame.mFlags & SHORT_SLOT) ? 0 : frame.mData1;
                wave_file_handler.addSample(sample_value);
                if (UpdateExportProgressAndCheckForCancel(i, num_frames) == true)
                {
                    wave_file_handler.close();
                    return;
                }
            }
        }
        UpdateExportProgressAndCheckForCancel(num_frames, num_frames);
        wave_file_handler.close();
    }
    else
    {
        // Standard PCM path for <=4 GiB exports
        PCMWaveFileHandler wave_file_handler(f, mSettings->mTdmFrameRate,
                                              mSettings->mSlotsPerFrame,
                                              mSettings->mDataBitsPerSlot);
        for (U64 i = 0; i < num_frames; i++)
        {
            Frame frame = GetFrame(i);
            if (frame.mType < num_slots_per_frame)
            {
                U64 sample_value = (frame.mFlags & SHORT_SLOT) ? 0 : frame.mData1;
                wave_file_handler.addSample(sample_value);
                if (UpdateExportProgressAndCheckForCancel(i, num_frames) == true)
                {
                    wave_file_handler.close();
                    return;
                }
            }
        }
        UpdateExportProgressAndCheckForCancel(num_frames, num_frames);
        wave_file_handler.close();
    }
}
```

**Key points:**
- The export loop body is identical in both branches — same `GetFrame`, same `SHORT_SLOT` silence, same `addSample`, same cancel check. The only difference is the handler type. This duplication is intentional: both handlers have the same interface but are different types, so a template or base class would add unnecessary complexity for a two-branch dispatch.
- The `num_slots_per_frame` variable is hoisted before the branch (used in both paths).
- The commented-out `PCMExtendedWaveFileHandler` line that currently exists in the PCM path should be removed — it is dead code.
- Preserve the existing code style: spaces inside parentheses for `if(...)`, Allman-style braces, same indentation.
- Add a brief comment above each branch: `// RF64 path for >4 GiB exports` and `// Standard PCM path for <=4 GiB exports`.
