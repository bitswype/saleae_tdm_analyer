In `src/TdmAnalyzerResults.cpp`, add a pre-export size check at the beginning of `GenerateWAV()`, before the `PCMWaveFileHandler` is constructed.

**Implementation:**

1. After `U64 num_frames = GetNumFrames();` (or equivalent frame count retrieval), compute the estimated WAV data size:

```cpp
// Compute bytes per channel based on data bit depth (same mapping as PCMWaveFileHandler ctor)
U32 bits = mSettings->mDataBitsPerSlot;
U32 bytes_per_channel;
if      ( bits <=  8 ) bytes_per_channel = 1;
else if ( bits <= 16 ) bytes_per_channel = 2;
else if ( bits <= 32 ) bytes_per_channel = 4;
else if ( bits <= 40 ) bytes_per_channel = 5;
else if ( bits <= 48 ) bytes_per_channel = 6;
else                   bytes_per_channel = 8;

U64 frame_size_bytes = ( U64 )bytes_per_channel * mSettings->mSlotsPerFrame;

// WAV RIFF data chunk size is a U32; max data payload is UINT32_MAX - 36 bytes
// (36 = fixed PCM header overhead before the data chunk).
// num_frames is a conservative upper bound — actual output may be smaller.
constexpr U64 WAV_MAX_DATA_BYTES = ( U64 )0xFFFFFFFF - 36ULL;
U64 estimated_data_bytes = num_frames * frame_size_bytes;
```

2. If `estimated_data_bytes > WAV_MAX_DATA_BYTES`, write a text warning to the output file and return early:

```cpp
if ( estimated_data_bytes > WAV_MAX_DATA_BYTES )
{
    std::ofstream f;
    f.open( file, std::ios::out );
    if ( f.is_open() )
    {
        f << "WAV export aborted: estimated output ("
          << ( estimated_data_bytes / ( 1024ULL * 1024 * 1024 ) )
          << " GiB) exceeds the 4 GiB WAV RIFF format limit.\n"
          << "Export as TXT/CSV instead and convert to WAV with an external tool.\n";
        f.close();
    }
    UpdateExportProgressAndCheckForCancel( num_frames, num_frames );
    return;
}
```

3. The rest of `GenerateWAV()` continues unchanged after this guard.

**Follow project formatting conventions:** Allman braces, spaces inside parentheses for function calls `( value )`, 4-space indentation. Match the existing code style in TdmAnalyzerResults.cpp.

**Note:** `<fstream>` is already included in TdmAnalyzerResults.cpp. `UINT32_MAX` / `0xFFFFFFFF` — use the hex literal with ULL suffix as shown to avoid any header dependency issues.
